# Работа по формам ТОРГ-12 и УПД — отложено (WIP)

> Заметка-заготовка: текущее состояние разработки унифицированных форм накладной.
> Кнопка «Скачать PDF» для накладной временно скрыта для всех пользователей,
> кроме тестового (телефон `+70000000002`), пока формы ТОРГ-12 и УПД не доработаны.

## Где выбирается формат (фронтенд)
- Файл: `src/components/app/DocumentModal.tsx`
- Состояние формата: `docFormat: "simple" | "torg12" | "upd"` (по умолчанию `simple`)
- Форматы ТОРГ-12/УПД доступны ТОЛЬКО для `docType === "invoice_note"` (накладная).
  Для акта всегда `simple`.
- Выбор формата — «Format sheet» (нижняя шторка) с опциями:
  - `simple` — Обычная накладная
  - `torg12` — ТОРГ-12 (унифицированная форма)
  - `upd` — УПД (универсальный передаточный документ)

## Бэкенд генерации PDF
- Файл: `backend/invoices/index.py`
- Endpoint принимает `action: "document"` для документов реализации (акт/накладная).
- Ключевые поля payload: `doc_type`, `doc_format`, `doc_date`, `doc_number`,
  `invoice_number`, `invoice_date`, `client_name`, `client_inn`, `client_kpp`,
  `client_ogrnip`, `client_address`, `items`, `total`.

### ТОРГ-12 — `build_torg12()`
Статус: базовая полная реализация макета.
- Ориентация: A4 landscape (альбомная).
- Шапка: грузоотправитель/грузополучатель/поставщик/плательщик, адреса,
  основание, коды ОКУД/ОКПО, номер и дата.
- Таблица товаров с графами унифицированной формы (№, наименование, код,
  ед. изм. + код ОКЕИ, упаковка, кол-во/места, масса брутто/нетто, цена,
  сумма без НДС, ставка+сумма НДС, сумма с НДС).
- Итоги: всего единиц, масса, сумма без/с НДС, сумма прописью (`rub_words()`).
- Подписи: гл. бухгалтер, отпуск груза, груз принял/получил.
- Реквизиты продавца: `full_name, inn, kpp, address, okpo, bank_name, bik,
  checking_account, corr_account, entity_type`.

### УПД — заглушка внутри `build_document()`
Статус: НЕ доработано. Сейчас используется макет простой накладной с изменённым
заголовком «УНИВЕРСАЛЬНЫЙ ПЕРЕДАТОЧНЫЙ ДОКУМЕНТ».

Что НЕ реализовано для УПД:
- Полная структура счёта-фактуры (номер СФ, налоговая ставка, сумма НДС).
- Блок «передаточный документ» (акт реализации).
- Двухчастный макет (счёт-фактура + передаточный документ), статусы 1/2.
- Совместимость с ЭДО, поля для подписей/штампов.

## Что нужно доработать перед включением кнопки для всех
1. УПД — полноценный макет (счёт-фактура + передаточный документ).
2. ТОРГ-12 — сверка граф/реквизитов с унифицированной формой, НДС по позициям,
   проверка на реальных данных.
3. Ставка НДС по позициям (`vat_rate`) на фронтенде (сейчас на бэке `no_vat`
   по умолчанию), UI выбора ставки.
4. После готовности — снять временное ограничение показа кнопки «Скачать PDF»
   для накладной (см. `DocumentModal.tsx`, условие с тестовым телефоном).

## Временное ограничение (текущее поведение)
- В `src/components/app/DocumentModal.tsx` скачивание PDF накладной доступно
  ВСЕМ (кнопка «Скачать PDF» на месте), но из выбора формы для обычных
  пользователей убраны ТОРГ-12 и УПД — остаётся только «Обычная накладная».
- Форматы ТОРГ-12 и УПД видны только тестовому пользователю
  (телефон `+70000000002`) — флаг `isTestUser`.
- Для обычных пользователей формат при сохранении и генерации PDF
  принудительно `simple`, шторка выбора формы не открывается.
- Чтобы вернуть формы всем — убрать условие `isTestUser` в местах:
  список форматов (Format sheet), `openPdf`, `doSave`.
